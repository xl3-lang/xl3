xl3 Excel-to-Excel transformation
Workflow Handoff 운영 Use English

TypeScript engine · workbook-defined workflow

workbook template으로 Excel 변환 엔진을 만드세요.

xl3는 개발자가 반복되는 Excel 변환 업무를 template workbook으로 정의하고, 비개발자는 raw Excel을 올려 완성된 workbook을 내려받는 흐름을 만들 수 있게 합니다.

Workflow model

엔진은 코드에, 업무 흐름은 workbook에 남깁니다.

개발자는 변환기를 한 번 만들고, 반복되는 업무 규칙과 source table 매핑, 레이아웃, 출력 모양은 아카이빙하고 인수인계할 수 있는 template workbook 안에 둡니다.

01

계약을 Excel 안에 정의합니다

`_config`, `source_table`, XTL 표현식으로 raw workbook이 보고서가 되는 규칙을 적습니다.

02

단순한 업로드 흐름으로 노출합니다

비개발자는 raw Excel과 승인된 템플릿을 선택하고 변환기를 실행합니다.

03

완성된 workbook을 생성합니다

값은 데이터에서 렌더링하고, 시트 구조와 숫자 형식, 스타일, 병합 셀은 유지합니다.

04

운영 규칙을 파일로 남깁니다

템플릿은 반복되는 Excel 업무가 어떻게 동작하는지 담은 인수인계용 산출물이 됩니다.

data.xlsx workbook이 원본 row를 공급합니다.
source_table의 컬럼명이 XTL source column이 됩니다.

개발자가 쓰는 이유

반복 보고서 로직을 일회성 스크립트 밖으로 꺼냅니다.

Python script, VBA macro, 서비스별 workflow로도 Excel 자동화는 가능합니다. 하지만 업무 규칙은 코드, 계정, 담당자의 기억에 흩어지기 쉽습니다.

xl3는 재사용 가능한 엔진과 workbook별 변환 계약을 분리합니다. 개발자는 TypeScript 통합을 관리하고, 반복되는 Excel 업무는 template workbook이라는 파일 단위로 전달할 수 있습니다.

운영 관점

자동화를 넘어서 인수인계 가능한 형태로 만듭니다.

실제로 변환기를 실행하는 사람은 코드를 읽지 않아도 됩니다. 필요한 것은 안정적인 변환기, 승인된 템플릿, 예측 가능한 결과 workbook입니다.

템플릿이 인수인계 문서입니다

보고서 규칙은 검토하고, 버전 관리하고, 보관하고, 다음 담당자에게 넘길 수 있는 workbook 파일에 남습니다.

template.xlsx

실무자에게 단순합니다

브라우저 페이지로 노출하면 raw 데이터를 올리고, 템플릿을 선택하고, 결과 workbook을 다운로드하면 됩니다.

raw.xlsx -> result.xlsx

엔진은 개발자가 관리합니다

배포, 검증, 서비스 통합은 코드에서 관리하고, 템플릿별 업무 규칙은 Excel 안에 둡니다.

convert(template, raw)
