Excel 변환 로직을, Excel 파일 안에.

xl3는 변환 규칙을 코드가 아니라 template.xlsx 안에 넣습니다. 비개발자도 직접 열어 읽고 고칠 수 있습니다 — IF, SUM, 열 참조처럼 평소 쓰던 Excel 문법 그대로니까요.

Workflow model

엔진은 코드에, 업무 흐름은 엑셀 파일에 남깁니다.

개발자는 변환기를 한 번 만들고, 반복되는 업무 규칙과 source table 매핑, 레이아웃, 출력 모양은 아카이빙하고 인수인계할 수 있는 엑셀 템플릿 안에 둡니다.

01

변환 규칙을 엑셀 안에 적습니다

`__config__`와 `source_table`, XTL 표현식으로 원본 데이터를 보고서로 바꾸는 규칙을 엑셀 안에 적어 둡니다.

02

단순한 업로드 화면으로 공개합니다

비개발자는 원본 Excel과 승인된 템플릿을 고르고 변환 버튼만 누르면 됩니다.

03

완성된 엑셀 파일을 생성합니다

값은 원본 데이터로 채우고, 시트 구조와 숫자 형식, 스타일, 병합 셀은 그대로 둡니다.

04

운영 규칙을 파일로 남깁니다

템플릿에 반복 업무의 동작 방식이 그대로 담겨, 그 자체가 인수인계 가능한 파일이 됩니다.

data.xlsx 템플릿이 원본 데이터의 모양을 지정합니다.
source_table의 컬럼명이 그대로 XTL의 컬럼 이름이 됩니다.

개발자가 쓰는 이유

반복 보고서 로직을 일회성 스크립트 밖으로 꺼냅니다.

Python 스크립트, VBA 매크로, 서비스별 자동화 도구로도 Excel 자동화는 가능합니다. 하지만 업무 규칙은 코드, 계정, 담당자의 기억에 흩어지기 쉽습니다.

xl3는 재사용 가능한 엔진과 엑셀 파일별 변환 규칙을 분리합니다. 개발자는 TypeScript 통합을 관리하고, 반복되는 Excel 업무는 엑셀 템플릿 파일 단위로 전달할 수 있습니다.

운영 관점

자동화를 넘어서 인수인계 가능한 형태로 만듭니다.

실제로 변환기를 실행하는 사람은 코드를 읽지 않아도 됩니다. 필요한 것은 안정적인 변환기, 승인된 템플릿, 예측 가능한 결과 엑셀 파일입니다.

템플릿이 인수인계 문서입니다

보고서 규칙은 검토하고, 버전 관리하고, 보관하고, 다음 담당자에게 넘길 수 있는 엑셀 파일에 남습니다.

template.xlsx

실무자에게 단순합니다

브라우저 페이지로 노출하면 raw 데이터를 올리고, 템플릿을 선택하고, 결과 엑셀 파일을 다운로드하면 됩니다.

raw.xlsx -> result.xlsx

엔진은 개발자가 관리합니다

배포, 검증, 서비스 통합은 코드에서 관리하고, 템플릿별 업무 규칙은 Excel 안에 둡니다.

convert(template, raw)
